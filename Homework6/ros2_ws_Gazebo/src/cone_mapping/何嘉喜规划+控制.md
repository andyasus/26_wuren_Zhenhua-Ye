# planning_control 包总结

## 做了什么

路径规划 + 车辆控制，订阅 `/vehicle_pose`，发布 `/cmd_vel` 让车沿着赛道跑。

## 文件结构

```
ros2_ws/src/planning_control/
├── planning_control/
│   ├── path_planner_node.py      # 路径规划
│   └── controller_node.py        # Pure Pursuit 控制
├── launch/
│   └── planning_control_launch.launch.py
├── test_path_control.py          # 离线测试
├── README.md
├── 笔记.md
└── 作业要求对照.md
```

## 路径规划

- 从 model.sdf 按赛道截面手动列出 12 个中线点
- 用 scipy 三次样条插值平滑成 200 个点
- 起点 (0, -15)，终点 (27, 12)，和赛道完全对齐
- 发布 `/path`（nav_msgs/Path）
- 预留了 `/track_map` 回调，等队友C建图好了可以切在线规划

## 车辆控制

- Pure Pursuit 纯追踪算法
- 动态前视距离：lookahead = 1.5 + 0.3 × 速度
- 弯道自动减速：曲率越大速度越慢
- 发布 `/cmd_vel`（geometry_msgs/Twist，linear.x=速度，angular.z=转向角）

## 参数

| 参数 | 值 | 说明 |
|------|:---:|------|
| wheelbase | 0.25 | 和仿真插件一致 |
| max_speed | 2.0 | 先设小一点 |
| min_speed | 0.5 | 弯道最低速度 |
| lookahead_base | 1.5 | 基础前视距离 |
| lookahead_gain | 0.3 | 速度增益 |
| max_steering | 0.6 | 最大转向角 34° |

## 接口

订阅：
- `/vehicle_pose`（geometry_msgs/PoseStamped）— 来自队友B
- `/track_map`（visualization_msgs/MarkerArray）— 来自队友C，可选

发布：
- `/path`（nav_msgs/Path）
- `/cmd_vel`（geometry_msgs/Twist）

## 踩过的坑

1. definition.md 写 `/cmd_ackermann`，实际仿真用 `/cmd_vel`，改过来了
2. 轴距 definition.md 写 0.3m，实际插件是 0.25m，按仿真来
3. 转弯部分左右锥桶数量不一样，直接取平均终点会偏，改成手动列中线点
4. 坐标系：world 东北天，base_link 前左上，算 Pure Pursuit 时要转坐标系

## 怎么跑

```bash
cd ros2_ws
colcon build --packages-select planning_control
source install/setup.bash
ros2 launch planning_control planning_control_launch.launch.py
```

离线测试（不用 ROS2）：

```bash
python test_path_control.py
```
